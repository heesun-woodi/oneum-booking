## 3. 테스트 순서 (최적화)

### 의존성 다이어그램

```
1단계: 회원가입 플로우
  └── 회원가입 신청 → 6-1 (관리자 알림)
  └── 가입 승인 → 1-2
  └── 가입 거부 → 1-3

2단계: 예약 플로우  
  └── 회원 예약 → 2-1
  └── 비회원 예약 → 2-2

3단계: 입금/취소 플로우 (2단계 예약 활용)
  └── 입금 확인 → 3-1
  └── 예약 취소 (입금 완료자) → 2-3 + 5-3

4단계: 크론 SMS (데이터 조작 또는 수동 호출)
  └── 입금 독촉 → 3-2 (미입금 예약 필요)
  └── 미입금 알림 → 5-2 (미입금 예약 필요)
  └── 내일 리마인더 (비회원) → 4-1
  └── 내일 리마인더 (회원) → 4-3
  └── 오늘 리마인더 → 4-2
```
